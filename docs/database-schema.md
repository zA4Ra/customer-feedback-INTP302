# Customer Feedback Sentiment Dashboard - Database Schema

## Overview

This database is used to store customer feedback and AI-generated sentiment analysis results. It supports a dashboard that allows business owners to view overall customer satisfaction trends in real time.

## Table: Feedback

This is the main table used to store all customer feedback entries along with sentiment analysis results generated by Azure AI Language.

### Purpose

- Store raw customer feedback text
- Store AI-generated sentiment labels
- Store extracted key phrases
- Track when feedback was submitted

## Field Descriptions

| Field Name  | Data Type                   | Description                                                 |
| ----------- | --------------------------- | ----------------------------------------------------------- |
| FeedbackID  | INT (Primary Key, Identity) | Unique identifier for each feedback entry                   |
| CommentText | NVARCHAR(MAX)               | The original customer feedback text                         |
| Sentiment   | VARCHAR(20)                 | AI-generated sentiment (positive, negative, neutral, mixed) |
| KeyPhrases  | NVARCHAR(MAX)               | Important keywords extracted from feedback                  |
| SubmittedAt | DATETIME                    | Timestamp when feedback was submitted                       |

## SQL Table Creation Script

```sql
CREATE TABLE Feedback (
    FeedbackID INT IDENTITY(1,1) PRIMARY KEY,
    CommentText NVARCHAR(MAX),
    Sentiment VARCHAR(20),
    KeyPhrases NVARCHAR(MAX),
    SubmittedAt DATETIME DEFAULT GETDATE()
);
```
